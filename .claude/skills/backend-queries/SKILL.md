## When to use this skill:

- When writing SQL queries or ORM query methods
- When implementing data fetching in API routes or services
- When preventing SQL injection with parameterized queries
- When optimizing queries to avoid N+1 problems
- When adding eager loading for related data
- When implementing database transactions
- When selecting specific columns instead of SELECT *
- When adding query timeouts or limits
- When caching expensive query results
- When working with Supabase client queries
- When using Prisma findMany, findUnique, etc.
