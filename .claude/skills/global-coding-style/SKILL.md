## When to use this skill:

- When naming variables, functions, classes, or files
- When structuring functions for readability
- When deciding function size and scope
- When removing unused code or imports
- When applying DRY principle (extracting common logic)
- When formatting code consistently
- When choosing between approaches for readability
- When refactoring existing code
- When reviewing code structure
- Always apply when writing or editing any code file
