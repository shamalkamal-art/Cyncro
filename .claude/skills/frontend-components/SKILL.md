Apply when working with files in /components/, creating .tsx or .jsx files, building reusable UI elements, or structuring component hierarchies.

## When to use this skill:

- When creating new React components
- When defining component props and TypeScript interfaces
- When deciding on component composition vs inheritance
- When implementing reusable UI patterns
- When managing component state (local vs lifted)
- When building design system components
- When naming components and their files
- When determining component boundaries and responsibilities
- When documenting component usage and props
- When working with files in `/components/` directories
- When creating .tsx, .jsx component files
